# Choix de l'Outil CI/CD

Nous avons décidé d'utiliser **GitHub Actions** pour notre pipeline CI/CD. Les raisons incluent :

- **Intégration native** : Avec GitHub, il est facile d'automatiser les workflows directement à partir de votre repository.
- **Flexibilité** : Permet de configurer des workflows complexes et de les adapter aux besoins spécifiques du projet.
- **Communauté et Support** : Une forte communauté, ce qui facilite la recherche de solutions et les bonnes pratiques.

GitHub Actions fournira une infrastructure robuste pour les tests automatisés, les déploiements continus et bien plus encore.
